The two measurements (why it's a range)

Conservative — code style

4.6%

Looks for the tell-tale style of AI tools (comment patterns, structure). Very reliable when it fires, but misses AI code that was edited into the author's own style — so it under-counts. This is a safe floor.

Generous — commit pattern

24.9%

Flags large blocks of code added in one go. Catches bulk AI generation, but also counts big hand-written files and one-time imports — so it over-counts. This is a generous ceiling.

🎯

For an exact figure, AI usage can be recorded at the moment code is written (a simple tag on each commit). Until then, this range is the most honest estimate — it brackets the true number rather than guessing a single one.

Method. Current code attributed to each author line-by-line via git blame; identities merged; automated bots excluded. AI estimated two ways: code-style fingerprinting (conservative) and large-addition commit analysis (generous). 4 commits carried explicit AI tags. Estimates carry inherent uncertainty and are intended as trends, not exact accounting.
